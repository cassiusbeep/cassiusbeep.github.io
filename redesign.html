responsive redesign: a case study

This case study aims to analyse the weaknesses of an existing webpage and help me develop the skills needed to create prototypes for a responsive redesign and eventually develop a working responsive website that adapts to different screen sizes.

part 1: identifying usability problems back to top

The webpage I've selected is Art Crimes at graffiti.org, an archive of graffiti and graffiti artists from around the world. It also hosts articles and other resources relating to graffiti culture and news. This site has been going a long time, and has actually split off from the historic version at graffitiarchiv.org — this site doesn't load for me but you may have better luck! The version I'm looking at is labelled the "future-facing" Art Crimes website.

Graffiti.org website home screen

My first task is to identify any significant choices or drawbacks in the design of this website, focusing on efficiency, learnability, and memorability.

    efficiency:
  • inefficient for keyboard/screen reader users: really long list to tab through - no shortcuts.
  • search bar at bottom might be inefficient.
  • the search bar is too short - doesn't even show the whole placeholder. annoying for longer search terms.
    learnability:
  • navigation is quite learnable for mouse users: all options are presented flatly (no nested menus, so no having to figure out which submenu your specific option is located under)
  • the main image is linked (not obviously) to a larger version of the image which is hard to tell without prior knowledge.
  • the high-level shape of the page doesn't really match natural reading patterns (the F-shape), so first-time visitors might have trouble orienting themselves.
  • it took me a second to realise the significance of the list of names on the right of the page (titled "March 2017"). it also wasn't clear that the "Previous months" list is connected.
    memorability:
  • it might be hard to remember the location of frequently used links since they're just in a long list rather than spatially distributed.
  • there's not much going on within the page so generally pretty memorable. i'd like to preserve this simplicity.

what is the conceptual model of graffiti.org?

Art Crimes aims to be a kind of hub or central navigation point for all things graffiti. This homepage should allow visitors to explore freely around artists and archives if they have a general interest, or quickly navigate to specific resources if they know what they want to find. There's also an element of trying to showcase different artists - maybe their art should be more accessible from the list, to encourage exploration and discovery?

    I also used WebAIM WAVE to detect possible accessibility problems. It highlighted that:

  • there isn't a clear heading hierarchy, with the largest heading being H2 and no subheadings below that.
  • the site uses spacer images without alt text.
  • their HTML file doesn't specify a language or is invalid.
  • they include a linked image (which I mention above) without alt text, so the link is "empty".
  • the page uses very small text in the footer for copyright information and acknowledgements.
  • the page layout is mainly dictated by layout tables, which I feel are kind of old school and might not be best practice for modern layouts.

    • I also separately noticed that the page lacks H1 headings because an images/logo is used as the main title. I understand the appeal for an art-focused website, but the accessibility and navigability can definitely be improved.

part 2: visual redesign back to top

Before creating high definition prototypes for my redesign across different platforms, I put together a visual design style guide that shows the main elements I'd be using. This includes important reusable components, my colour palette, and typography choices.

    This guide includes the components I intend to use throughout the redesign, including variants for different states:
  • Responsive components like hoverable links, interactive buttons - these make behaviours to user actions more predictable.
  • Collapsed menu into dropdowns to simplify navigation.
  • Kept the colour scheme consistent (minimal -- there's lot of colour in the graffiti!)
  • Updated typefaces to make them legible but still stylistic
  • Removed uses of images as tites or spacers, especially without alt text.
  • Removed the separate style/colour for visited links (no more purple links) based on user feedback that it's confusing/unhelpful
  • Queries the user's ability to hover to remove some buttons specifically on touchscreens
  • Slightly enlarge images on hover to hint at the click-to-maximise feature

Since the existing site does not scale at all in response to changing font or screen sizes, I introduced grid layouts (which can stack elements on narrower screens) and variably sized image elements to improve visibility.

Not pictured in the annotations for clarity is the overarching grid layout of the whole body of the page, which would be a single column. I use this to limit the width of the centrally scrolling page contents in case of ultra-wide screens.

part 3: responsive redesign back to top

Using my critiques from the earlier parts and the style guide/prototypes I built in Figma, I went on to build a working, responsive version of my redesigned webpage for Graffiti.org. The links in my prototype are hooked up and will lead you to pages in the real website, so feel free to explore and see how the navigation feels compared to the original homepage.

Graffiti.org Redesign: Homepage

    Here are some of the choices I made to improve usability and accessibility:
  • Implemented variable image sizes and grid configurations for different screen/font sizes
  • Re-organised headings and content to fit user expectations and introduce the site better
  • Made menu navigation more logical and streamlined
  • Used tab index and JS event listeners to improve keyboard navigation (try tabbing through the navbar!)
  • Added alt-text to images, removed spacer images, and used a consistent header hierarchy.

    • I think my design is pretty effective at preserving the visual brand of the Art Crimes organisation while improving the overall user experience and drawing people in to explore the archives.

I had a fun time redesigning this webpage and thinking about how simple design choices and responsiveness testing can improve the end user experience. Going through critique was also a great opportunity to identify blind spots in my process. Maybe one day Graffiti.org will use some of these ideas for a redesign!
